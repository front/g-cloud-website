Why?

Code once, use everywhere: As Gutenberg blocks are CMS agnostic, we want to provide an ecosystem all systems can connect to.

One plugin: Installing an actual plugin/module for each block or block collection easily leads to managing lots of code that you don’t really need. It’s like having one app for each website you visit, instead of just installing a browser. With the block manager, you have a growing library at your fingertips, without the bloat.

Share your code

  1. Create your block
  2. Publish it to NPM and tag it “gutenberg-cloud”
  3. Make sure to add the custom tag “screenshot” in your package.json
  4. Wait a few hours for NPM to update, and your blocks are automatically available
